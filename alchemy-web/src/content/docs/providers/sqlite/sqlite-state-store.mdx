## Engine Selection

### Automatic Detection

SQLiteStateStore automatically detects available SQLite engines in this order:

1. **Bun SQLite** (`bun:sqlite`) - Built into Bun runtime
2. **libSQL** (`@libsql/client`) - Turso-compatible SQLite
3. **better-sqlite3** - Node.js native SQLite binding

### Explicit Engine Selection

Force a specific SQLite engine:

  </TabItem>
  <TabItem label="libSQL">

## Customization

### Custom Database Location

Change the database file location:

```typescript
const app = await alchemy("my-app", {
  stateStore: (scope) => new SQLiteStateStore(scope, {
    filename: "./data/my-app-state.db"
  })
});
```

### Environment Variable Configuration

Use environment variables for flexible configuration:

```sh
# .env
ALCHEMY_STATE_FILE=./data/production-state.sqlite
```

```typescript
// alchemy.run.ts - automatically uses ALCHEMY_STATE_FILE
const app = await alchemy("my-app", {
  stateStore: (scope) => new SQLiteStateStore(scope)
});
```

### Retain Database

Keep the database file after the state store is destroyed:

```typescript
const app = await alchemy("my-app", {
  stateStore: (scope) => new SQLiteStateStore(scope, {
    retain: true, // Database persists after destruction
    filename: "./persistent-state.db"
  })
});
```

## Environment-Specific Configuration

### Development vs Production

Use different configurations for different environments:

```typescript
const isDev = process.env.NODE_ENV === "development";

const app = await alchemy("my-app", {
  stateStore: (scope) => new SQLiteStateStore(scope, {
    filename: isDev 
      ? ".alchemy/dev-state.sqlite"
      : "/var/lib/myapp/prod-state.sqlite",
    retain: !isDev // Only retain in production
  })
});
```

### Per-Stage Databases

Use separate databases for different deployment stages:

```typescript
const stage = process.env.STAGE ?? "dev";

const app = await alchemy("my-app", {
  stage,
  stateStore: (scope) => new SQLiteStateStore(scope, {
    filename: `.alchemy/state-${stage}.sqlite`,
    retain: stage === "prod"
  })
});
```

## Remote SQLite with libSQL

### Turso Database

Connect to a remote Turso database:

```typescript
const app = await alchemy("my-app", {
  stateStore: (scope) => new SQLiteStateStore(scope, {
    engine: "libsql",
    url: process.env.TURSO_DATABASE_URL,
    authToken: process.env.TURSO_AUTH_TOKEN
  })
});
```

### Local Replica with Sync

Use a local replica that syncs with remote:

```typescript
const app = await alchemy("my-app", {
  stateStore: (scope) => new SQLiteStateStore(scope, {
    engine: "libsql",
    url: process.env.TURSO_DATABASE_URL,
    authToken: process.env.TURSO_AUTH_TOKEN,
    syncUrl: "file:./local-replica.db",
    syncInterval: 60 // Sync every 60 seconds
  })
});
```
## Performance Considerations

### Engine Performance

- **Bun SQLite**: Fastest for Bun runtime, zero additional dependencies
- **better-sqlite3**: Best Node.js performance, synchronous API
- **libSQL**: Best for remote/distributed scenarios, async API

### Database Optimization

```typescript
const app = await alchemy("my-app", {
  stateStore: (scope) => new SQLiteStateStore(scope, {
    engine: "better-sqlite3",
    // Optimize for performance
    timeout: 10000,
    verbose: undefined, // Disable logging in production
    pragma: {
      journal_mode: "WAL",
      synchronous: "NORMAL",
      cache_size: -64000 // 64MB cache
    }
  })
});
```

## Troubleshooting

### Missing Dependencies

If you see dependency errors:

```
[SQLiteStateStore] Missing peer dependencies: drizzle-orm, better-sqlite3
```

Install the required packages:

```sh
bun add drizzle-orm better-sqlite3
```

### Permission Errors

Ensure the directory is writable:

```sh
mkdir -p .alchemy
chmod 755 .alchemy
```
